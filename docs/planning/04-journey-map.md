## 🅰 김총대(방장) 저니맵

> 페르소나 요약: 28세 스타트업 백엔드 개발자. 팀 점심을 주 5회 "총대" 메고 결정하다 지침. JTBD: 후보 찾는 노동 없이 3분 안에 팀 합의를 만든다.

| 단계 | 사용자 행동 | 속마음(thought) | 감정 | 터치포인트 | 페인포인트 | 모디의 기회(기능) |
|---|---|---|---|---|---|---|
| **1. 트리거** | 팀 단톡에 "오늘 뭐 먹어요?" 올라옴. 아무도 대답 안 함. 결국 총대가 나선다 | "또 나야? 매번 내가 정해야 해?" | 😩 의무감·피로 | 카카오톡 단톡방 | 후보 수집이 늘 본인 노동. "아무거나" 반응에 지침 | — (트리거 단계, 모디 진입 이전) |
| **2. 랜딩 → 방 생성 폼** | 모디 링크를 열어 `/create` 폼에 조건 입력: 모임명·위치(강남역)·인원(5명)·예산(1만원)·타이머(3분) | "이거 얼마나 걸리지? 복잡하면 안 쓸 것 같은데…" | 🤔 기대반 우려반 | 모디 생성 폼(`/create`) | 새 도구 학습 비용 우려. 입력 필드가 많으면 이탈 | 폼 간결화(5개 필드 이하), 기본값 채움, 진행 단계 표시 없음(한 화면 완결) |
| **3. 후보 즉시 생성 ★아하①** | "생성하기" 클릭 → 1.5초 "후보 뽑는 중" 연출 → **후보 5개 카드가 3초 안에** 화면에 등장 | "어, 내가 검색 안 해도 되네? 이거 진짜 뜨잖아!" | 😀 놀람·해방감 | 방 대기화면(`/room/[id]`, waiting) — 후보 카드 5개 | 아하① 이전의 빈 화면 불안(빈 후보 뜨면 아하 즉사) | **아하①** 보장: mock 폴백 5개 필수, TTV ≤3s. "안 가본 집" 배지로 신선함 강조 |
| **4. 초대·팀 모음** | [URL 복사] 버튼 클릭 → 단톡에 링크 공유. 팀원 하나씩 입장하는 걸 `👥 N명 모이는 중` 카운트로 확인 | "애들이 들어오고 있네. 아직 2명. 더 기다릴까?" | 🙂 기대감·약간의 조바심 | 방 대기화면의 인원 카운터(실시간 1s 폴링), 카카오톡 | 팀원이 안 들어오면 방장 혼자 허공 투표 위기 | 인원 카운터 실시간 증가 시각화("새로 들어왔어요!"), 링크 재공유 버튼 접근 용이 |
| **5. 투표 시작·관찰** | 팀원 4명 확인 후 [투표 시작] 클릭 → 타이머 3분 가동. 본인도 후보에 👍/👎 투표. 남의 표가 차오르는 걸 실시간으로 봄 | "다들 잘 누르고 있네. 근데 저 집은 왜 저렇게 표가 없지…?" | 😌 참여감·관찰 재미 | 투표화면(`/room/[id]`, voting): 후보 카드, 👍/👎 버튼, 타이머, 집계 숫자 | 표가 편향되면 "우리 팀은 늘 같은 거"가 반복될 것 같은 막연한 우려 | 낙관적 즉시 반영(내 표) + 1s 폴링(남의 표)으로 생동감. 타이머 막판 10초 빨강 깜빡으로 클라이맥스 연출 |
| **6. 결정 확정** | 타이머 0 (또는 [지금 마감]) → 자동 잠김 → 🎉 컨페티 + 1등 카드 zoom-in 전원 표시 | "이걸로 결정됐다! 내가 고른 것도 아니고, 다 같이 정한 거잖아." | 😄 후련함·성취감 | 결과화면(`/room/[id]`, decided): 컨페티, 1등 카드, "결정됐어요!" | 동점 교착 우려(이미 seq 선착순으로 해결) | 1등 카드 강조 zoom-in 연출, 공정성 체감("다수결로 정해짐") |
| **7. 이동·식사** | 결과 카드 캡처하거나 닉네임 기억해 팀과 이동 | "다음 주에도 이거 쓰면 되겠다. 근데 내가 입력 또 다 해야 하나?" | 😊 만족, 약간의 재사용 고민 | 결과화면, 카카오톡(공유), 오프라인 식당 | 길찾기 없음(현 MVP 비범위). 식당까지 가는 과정은 모디 밖 | (P1) 지도 연동·길찾기. 현재는 식당명 클립보드 복사로 최소 대응 |
| **8. 사후·기록** | [처음으로] 클릭 혹은 창 닫음. 오늘 결정 어딘가에 남나? 별도 기록은 안 함 | "저번에 어디 갔었는지 까먹겠는데…" | 😐 아쉬움(기록 없음) | 랜딩(`/`) — 결정 이력 없음(P0 MVP) | 이력 0에서 다음에 또 처음부터 해야 함 = 🅲 리텐션 루프 부재 | (P1) 결정 이력 저장 → 개인화 추천. ⑤ 리텐션 KPI의 주인공 |

**🅰 감정선:** `😩(트리거) → 🤔(폼) → 😀(후보즉시=아하①) → 🙂(초대) → 😌(투표관찰) → 😄(확정) → 😊(식사) → 😐(사후)`

## 🅲 박모임(운영자, P1) 미니 저니맵

> 페르소나 요약: 31세 직장인. 주말 동호회·평일 스터디를 주 2회 운영. 매번 식당 후보를 새로 찾다 지침. JTBD: 기록 기반 자동 추천으로 반복 운영 피로를 없앤다.

### 1회차 — 첫 경험 (=🅰+🅱와 동일, 추가 가치 없음)

| 단계 | 행동 | 감정 |
|---|---|---|
| 방 생성 | 동호회 모임 전날 모디로 방 생성, 링크 밴드에 공유 | 😀 (아하①: 후보 즉시 생성) |
| 투표·결정 | 멤버들 입장·투표 → 3분 안에 결정 | 😄 (효율적 운영 쾌감) |
| 사후 | 결과 확인. "다음에도 쓸 것 같다"고 생각 | 🙂 (만족, 재사용 의향 생성) |

### 2회차+ — 반복 루프에서 아하 (🅲만의 아하)

| 단계 | 행동 | 속마음 | 감정 | 모디의 기회 |
|---|---|---|---|---|
| **재방문** | 다음 주 모임 전날 모디 재접속 | "저번엔 한식 갔으니까 이번엔 다른 거 추천해주겠지?" | 🙂 기대감 | (P1) 로그인·그룹 계정으로 이력 조회 |
| **기록 기반 추천 ★🅲 아하** | 저번 결정(한식 갈비탕) 자동 제외 + **"이번엔 다른 거" 후보 5개** 즉시 노출 | "어, 지난번 간 데 빠지고 새 것만 나오네. 얘가 우리 모임을 기억하네!" | 😀 **개인화 아하** | (P1) 결정 이력 저장 → 비노출 필터 + 새로운 후보 우선 추천 |
| **즉시 투표 시작** | 추천 검토 후 링크 바로 공유 → 투표 시작 | "뭐 찾아볼 것도 없네. 그냥 시작하면 되잖아" | 😄 운영 효능감 | 후보 편집 최소화(추천 신뢰도 높이기) |
| **결정 → 이력 누적** | 결과 저장됨 → 다음 회차의 필터 데이터로 활용 | — | 🔄 (리텐션 루프 강화) | 이력 누적 → 개인화 품질 향상 → 재사용률↑ |

### 리텐션 루프 (⑤ Retention KPI와 연결)

```
기록 저장(decided 이력)
       ↓
개인화 추천("저번엔 한식 → 이번엔 양식 추천")
       ↓
더 빠른 결정(후보 재검토 시간 단축)
       ↓
재사용(방 생성 반복) ← 이게 ⑤ Retention 지표의 핵심 분자
       ↓
기록 추가 저장 → 루프 재시작
```

> 참고: 리텐션 루프는 🅲가 P1인 이유이기도 하다. P0 MVP는 루프의 첫 단계(기록 저장)만 자리를 열어두고, 개인화 추천은 구현을 미룬다. ⑤ KPI의 재방문율·기록 이력 활용률이 이 루프의 건강지표다.

## 감정선 종합(Emotional Arc) — 🅰 vs 🅱 겹치기

아래 텍스트 그래프는 세로축 = 감정 레벨(높을수록 긍정), 가로축 = 공통 시간선.

```
감정  │
레벨  │
 😄  │         ★아하①                        😄확정
 😀  │          😀(🅰)    🙂🙂       😄    😄결과
 🙂  │  🤔        🙂초대  ─── 😌 ─── 확정 ─── 이아무
 😐  │    └──→ 닉네임 😀(🅱)아하②
 🙂  │                   ↑투표시작
 😐  │ 😐링크받음
 😩  │ 😩(🅰 트리거)
      │
      └───────────────────────────────────────────────▶ 시간
        트리거  폼/링크  입장   대기   투표시작  집계   결정확정

── 🅰 김총대 ──────────────────────────────────────────
── ─ ─ 🅱 이아무 ─────────────────────────────────────

[골(페인)] 🅰 트리거: 😩 "또 내가?"
[골(페인)] 🅱 링크 받음: 😐 "가입 시키겠지"
[정점(아하①)] 🅰 후보 즉시 생성: 😀 "검색 안 해도 되네"   ← TTV ≤3s (②)
[정점(아하②)] 🅱 닉네임 입장+투표: 😀 "가입 없이 벌써 같이 정하네" ← TTV ≤10s (②)
[공동 정점] 결정 확정: 😄 양쪽 모두 최고조 — 그룹 합의의 카타르시스
```

핵심 관찰:
- **아하①·아하② 모두 곡선의 가장 가파른 상승 구간**이다. 이 구간이 흐릿해지면(빈 후보, 가입 벽) 전체 감정선이 평탄해지거나 꺾인다.
- 🅱는 😐에서 시작해 아하② 한 방으로 😄까지 점프하는 구조 → 진입 경험의 질이 전부.
- 🅰는 트리거(😩) 직후부터 아하①까지의 **회복 속도**가 제품 가치의 체감 크기를 결정한다.
