# ⑥ 설문조사 설계 — 모디(MODI)

> 작성 관점: Product Manager + UX Researcher.
> 목적: 컨셉 수요를 사전에 검증(설문 A)하고, 데모·사용 직후 만족·재사용 의향을 측정(설문 B)해 ⑤ KPI 목표치(HEART Happiness ≥ 4.0 / 재사용의향 ≥ 60%)와 ② 아하 가설(H1·H2)을 데이터로 뒷받침한다.
> **리딩 질문 회피 원칙:** 모든 문항은 모디·경쟁 서비스의 장점을 먼저 제시하지 않는다. 상황 묘사 → 행동 서술 → 평가 순으로 진행하고, "~가 좋지 않나요?"식 유도 어구 금지.

## 1. 리서치 목표 & 가설

### 1-1. 리서치 목표

| # | 목표 | 유형 |
|---|---|---|
| RO1 | 그룹 점심 결정이 충분히 빈번하고 고통스러운 문제인지 확인 | 컨셉 수요 |
| RO2 | 첫 방에서 결정 완료(decided) 경험이 재사용 의향을 높이는지 검증 | 행동-결과 인과 |
| RO3 | 방당 실투표 참여자 수가 만족·재사용 의향과 상관관계가 있는지 검증 | 참여 밀도-만족 |
| RO4 | 아하①(후보 즉시 노출) · 아하②(가입 없이 투표)가 체감되는지 확인 | UX 품질 |
| RO5 | 가입 거부감, 개인정보 우려 등 채택 장벽의 실질적 크기 파악 | 채택 장벽 |

### 1-2. 검증 가설 (검증 가능한 형태)

| 가설 | 내용 | 검증 문항 | 합격 기준 |
|---|---|---|---|
| **H1** | *첫 방에서 결정 완료(decided)까지 경험한 그룹은 재사용 의향이 유의하게 높다.* (②-§5에서) | B3, B4, B8 | decided 경험 그룹 vs 미경험 그룹 간 재사용 의향(B3) 평균 ≥ 0.5점 차이 |
| **H2** | *방당 실투표 참여자 ≥ 3명이면 만족·재사용↑.* (②-§5에서) | B1, B2, B3, B9 | 참여자 3명+ 방 vs 2명 방의 만족(B1) 평균 ≥ 0.3점 차이 |
| **H3** | *그룹 점심 결정은 충분히 자주·고통스러운 문제다* (컨셉 수요 가설, 신규) | A1, A2, A3 | "주 3회 이상 그룹 결정" & "현재 방식 불만" ≥ 4점인 응답자 비율 ≥ 60% |

> 가설 번호 기원: H1·H2는 ② 아하 모먼트 §5 "매직 넘버 가설"에서 직접 인용. H3은 본 문서에서 신설.

### 1-3. 가설 × 문항 매핑

```
H1 ──┬── B3 (재사용 의향 NPS-lite)
     ├── B4 (결정 완료 여부 — 사실 확인)
     └── B8 (개방형: 결정됐을 때 어떤 느낌?)

H2 ──┬── B1 (전반 만족도)
     ├── B2 (참여자 수 체감 — "다 같이 정하는 느낌")
     ├── B3 (재사용 의향)
     └── B9 (불편했던 점 — 인원 미달 체감 여부 포착)

H3 ──┬── A1 (그룹 점심 결정 빈도)
     ├── A2 (현재 방식 불만도)
     └── A3 (결정에 걸리는 시간 불만)
```

## 2. 설문 A — 컨셉 검증 (사전, 잠재 사용자 대상)

> 대상: 모디를 아직 써보지 않은 직장인·대학생 20~30명.
> 목적: H3(문제 빈도·심각도)와 채택 장벽(가입 거부감·대안 현황·사용 의향) 검증.
> 소요 시간: 약 5~7분.

### 문항 뱅크

| 번호 | 문항 | 유형 | 측정 목표 |
|---|---|---|---|
| A1 | **일주일 동안 직장 동료·친구·학과 사람들과 점심 또는 식사 장소를 함께 정하는 일이 몇 번 정도 있나요?** ① 거의 없음(주 1회 미만) ② 주 1~2회 ③ 주 3~4회 ④ 거의 매일(주 5회 이상) | 객관식(단일) | H3 빈도 |
| A2 | **"어디 갈지 다 같이 정하는 과정"이 얼마나 불편하거나 번거롭게 느껴지나요?** (1 = 전혀 불편하지 않다 / 5 = 매우 불편하다) | 5점 리커트 | H3 심각도 |
| A3 | **보통 그 자리에서 식사 장소가 정해지기까지 얼마나 걸리나요?** ① 1분 이내 ② 2~5분 ③ 6~10분 ④ 10분 이상 / 결국 흐지부지됨 | 객관식(단일) | H3 소요시간 |
| A4 | **현재는 주로 어떻게 식사 장소를 정하나요? (해당되는 것 모두 선택)** □ 가장 큰 목소리 한 명이 정함 □ 카카오톡 투표 □ 네이버지도 공유 링크 □ "아무거나"로 수렴 □ 로테이션(순번) □ 기타: _______ | 다중선택 | 현재 대안 파악 |
| A5 | **지금 쓰는 방식에서 가장 불편한 점은 무엇인가요?** (자유응답) | 개방형 | 페인포인트 심화 |
| A6 | **이런 서비스가 있다고 상상해보세요: 위치와 예산을 넣으면 주변 식당 후보 5개가 자동으로 뜨고, 팀원들이 링크만 눌러 가입 없이 좋아요/싫어요를 눌러 함께 고르는 웹 서비스.** 이 서비스를 실제 상황에서 써볼 의향이 얼마나 있나요? (1 = 전혀 없다 / 5 = 무조건 써볼 것 같다) | 5점 리커트 | 컨셉 사용 의향 |
| A7 | **위 서비스에 가입이나 회원 계정이 필요하다면 실제로 쓸 의향이 어떻게 바뀌나요?** ① 상관없다 ② 조금 덜 쓸 것 같다 ③ 많이 덜 쓸 것 같다 ④ 거의 안 쓸 것 같다 | 객관식(단일) | 채택 장벽(가입) |
| A8 | **링크만 눌러서 가입 없이 참여할 수 있다면, 내 개인정보(이름·번호 등)를 따로 입력하지 않아도 되는 것이 얼마나 중요한가요?** (1 = 전혀 중요하지 않다 / 5 = 매우 중요하다) | 5점 리커트 | 채택 장벽(프라이버시) |
| A9 | **이런 도구를 팀 단체 카카오톡에 공유하면 팀원들이 실제로 사용할 것 같나요?** (1 = 거의 안 쓸 것 같다 / 5 = 대부분 쓸 것 같다) | 5점 리커트 | 사회적 전파 가능성 |
| A10 | **지금까지 설명한 서비스와 비슷한 것을 써본 적 있나요?** ① 없다 ② 카카오톡 투표로 비슷하게 해봤다 ③ 다른 앱/서비스 써봤다(이름: _____) | 객관식(단일) | 대안 인식 |
| A11 | **한 가지만 더 물어볼게요. 이 서비스가 "여기 가자" 한 곳을 딱 결정해주지 않고, 후보 중 투표해서 고르는 방식이에요. 이 점이 마음에 드나요, 아닌가요?** (1 = 차라리 하나만 알려주는 게 낫다 / 5 = 같이 고르는 방식이 좋다) | 5점 리커트 | 협의 방식 선호 |
| A12 | **이 서비스에 대해 궁금한 점이나, 있으면 더 좋겠다고 생각하는 기능이 있다면 자유롭게 써주세요.** (자유응답) | 개방형 | 미발견 니즈 |

> 리딩 질문 회피 원칙 적용 내역:
> - A6: "좋지 않나요?" 대신 "써볼 의향이 얼마나 있나요?"로 중립 표현.
> - A7: 가입 장벽을 "나쁜 것"으로 프레이밍하지 않고 변화 방향 서술.
> - A11: 양극 라벨(1=한 곳만 / 5=같이 고르기)으로 방향성 숨김.

## 3. 설문 B — 사용성·만족 (사후, 데모/사용 직후)

> 대상: 모디를 직접 써본 데모 참가자·테스터 5~10명.
> 목적: H1·H2 검증, ⑤ HEART Happiness 목표(만족 ≥ 4.0 / 재사용의향 ≥ 60%) 측정, 아하① ② 체감 확인.
> 소요 시간: 약 4~6분. 반드시 사용 직후(30분 이내) 배포.

### 문항 뱅크

| 번호 | 문항 | 유형 | 측정 목표 |
|---|---|---|---|
| B1 | **방금 모디로 식사 장소를 정해본 전반적인 경험이 어떠셨나요?** (1 = 매우 불만족 / 5 = 매우 만족) | 5점 리커트 | HEART Happiness — ⑤ 목표 ≥ 4.0 |
| B2 | **"다 같이 정하고 있다"는 느낌이 얼마나 들었나요?** (1 = 전혀 없었다 / 5 = 매우 강하게 느꼈다) | 5점 리커트 | H2 참여 밀도 체감 |
| B3 | **친구·동료에게 이 서비스를 권유할 의향은 0~10점 중 몇 점인가요?** (0 = 절대 안 권유 / 10 = 무조건 권유) | NPS-lite 0~10 | ⑤ 재사용의향 ≥ 60%(추천의향 ≥ 7점 비율로 측정) |
| B4 | **오늘 경험에서 식사 장소가 최종적으로 결정되었나요?** ① 네, 한 곳으로 정해졌어요 ② 아니요, 결정 전에 끝났어요 ③ 잘 모르겠어요 | 객관식(단일) | H1 결정 완료 여부 확인(⑤ NSM 연결) |
| B5 | **"후보 식당이 바로 떠서 직접 찾아볼 필요가 없었다"는 말이 오늘 경험과 얼마나 일치하나요?** (1 = 전혀 그렇지 않다 / 5 = 완전히 그랬다) | 5점 리커트 | 아하① 체감 — ② §1 |
| B6 | **"가입 없이 링크만 눌러서 바로 투표할 수 있었다"는 말이 오늘 경험과 얼마나 일치하나요?** (1 = 전혀 그렇지 않다 / 5 = 완전히 그랬다) | 5점 리커트 | 아하② 체감 — ② §1 |
| B7 | **식사 장소를 정하는 데 걸린 시간이 평소보다 빠르다고 느껴졌나요?** (1 = 오히려 더 오래 걸린 것 같다 / 5 = 훨씬 빨랐다) | 5점 리커트 | 결정 속도 체감 — ⑤ Task success ≤ 3분 |
| B8 | **오늘 사용하면서 가장 좋았던 점이 무엇인지 자유롭게 써주세요.** (자유응답) | 개방형 | 아하 체감 질적 수집, H1 결정 완료 체감 |
| B9 | **반대로 가장 불편하거나 아쉬웠던 점은 무엇인가요?** (자유응답) | 개방형 | 개선 우선순위, 아하 저해 요인 포착 |
| B10 | **다음에 비슷한 상황이 생기면 모디를 다시 쓸 의향이 있나요?** (1 = 전혀 없다 / 5 = 무조건 다시 쓸 것 같다) | 5점 리커트 | ⑤ HEART Retention(의향 대체) |
| B11 | **오늘 함께 사용한 사람이 총 몇 명이었나요? (본인 포함)** ① 2명 ② 3명 ③ 4~5명 ④ 6명 이상 | 객관식(단일) | H2 실투표 참여자 수 파악 |
| B12 | **서비스 전반이 얼마나 쉽고 직관적이었나요?** (1 = 매우 어려웠다 / 5 = 매우 쉬웠다) | 5점 리커트 | 사용성(SUS 축약 대체) |

> SUS 전체 10문항 대신 B1·B12로 핵심 사용성 신호를 축약 포착. 소표본 환경에서 응답 부담 최소화.
> B3 NPS-lite 계산: 추천의향 7~10점 응답자 / 전체 응답자 × 100 = 재사용 추천율. ⑤ 목표 ≥ 60%.

## 4. 데모데이 30초 마이크로 설문

> 대상: 데모데이 현장 관람객 (QR코드 / 단축 URL 배포, 구글폼).
> 목적: 즉답형 4문항으로 ⑤ Happiness·사용의향 데이터 최소 수집.
> 소요 시간: 30초~1분.

| 번호 | 문항 | 유형 |
|---|---|---|
| M1 | **오늘 모디를 써봤을 때 얼마나 만족스러웠나요?** (1~5점) | 5점 리커트 |
| M2 | **"후보가 바로 떠서 편했다 / 가입 없이 바로 눌렀다" — 이 중 더 기억에 남는 것은?** ① 후보가 바로 뜬 것 ② 가입 없이 바로 투표 ③ 둘 다 ④ 잘 모르겠다 | 객관식(단일) |
| M3 | **팀 점심 결정할 때 이런 도구가 있으면 쓰겠다고 느꼈나요?** (1 = 아니다 / 5 = 무조건 쓰겠다) | 5점 리커트 |
| M4 | **한 가지만: 오늘 경험에서 가장 기억에 남는 점은?** (자유응답, 한 문장) | 개방형 |

> QR 링크는 구글폼 단축 URL, 슬라이드·포스터에 동시 배포. 문항 4개 초과 금지(현장 이탈 방지).

## 5. 측정·척도 원칙

### 5-1. 5점 리커트 라벨 통일

모든 5점 리커트는 아래 기준을 준수한다.

| 점수 | 긍정 방향 라벨 | 부정 방향 라벨 |
|---|---|---|
| 1 | 전혀 그렇지 않다 | 매우 불만족 |
| 2 | 그렇지 않다 | 불만족 |
| 3 | 보통이다 | 보통이다 |
| 4 | 그렇다 | 만족 |
| 5 | 매우 그렇다 | 매우 만족 |

- **양극 회피:** "나쁘지 않다 / 꽤 좋다"처럼 비대칭 라벨 금지.
- **단일 개념 1문항 원칙:** "빠르고 만족스러웠다"처럼 2개 개념을 1문항에 묶지 않는다(예: B7·B1 분리).
- **역채점 문항 없음:** 소표본·비전문 응답자 대상이라 역채점 혼란 유발 최소화.

### 5-2. NPS-lite 계산식

```
추천율(%) = (B3 ≥ 7점 응답 수) / (전체 응답 수) × 100
```

- 0~6점: 비추천(Detractor), 7~8점: 수동적(Passive), 9~10점: 추천(Promoter).
- 데모 목표: 추천율 ≥ 60% (⑤ HEART Happiness 재사용의향 목표와 동일 기준).
- **소표본 한계 명시:** N < 30에서 NPS 공식(Promoter% − Detractor%)은 통계적 신뢰 구간이 넓다. 본 데모에서는 "추천율(≥7점 비율)"만 보고하고 NPS 점수(-100~100)는 해석 보류.

### 5-3. 객관성 확보 원칙

| 편향 유형 | 대응 방안 |
|---|---|
| 자기선택 편향 | 설문 A는 특정 기술 관심자 채널(개발자 커뮤니티 등) 집중 모집 지양; 타깃(직장인·대학생) 일반 채널 우선 |
| 사회적 바람직성 편향 | "좋은 것 같다"로 답하게 만드는 긍정 문항 앞배치 금지 → A 설문은 문제 인식 문항(A1~A3)으로 시작 |
| 소표본 한계 | 결론 표현은 "~% 응답" 수준으로 제한; "통계적으로 유의하다" 주장 금지 |
| 순서 효과 | 사용성·만족 문항(B1~B3)을 개방형(B8·B9) 전에 배치해 긍정/부정 앵커링 분리 |

## 7. 분석 계획

### 7-1. 가설별 합격 기준

| 가설 | 측정 방법 | 합격 기준 | 연결 KPI (⑤) |

| **H1** | B4 × B3 그룹 비교 | B4=① (결정 완료) 그룹의 B3 평균 − B4=②③ 그룹의 B3 평균 ≥ 0.5 (5점 척도 기준) | HEART Retention 의향 |
| **H2** | B11 × B1 그룹 비교 | B11=② 이상 (3명+) 그룹의 B1 평균 ≥ B11=① (2명) 그룹의 B1 평균 + 0.3 | HEART Happiness + Engagement |

### 7-3. 정량 집계 절차

1. 구글폼 CSV 내보내기 → 구글 스프레드시트.
