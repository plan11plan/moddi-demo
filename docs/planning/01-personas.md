# ① 페르소나 — 모디(MODI)

> 작성 관점: Product Manager + Product Designer. 근거: 4조 PRD + 배포된 MVP(`moddi-demo.vercel.app`, 강남역 점심 데모).
> 목적: 누구의 어떤 문제를 푸는지 못 박아, 이후 아하·플로우·KPI·설문이 같은 사람을 향하게 한다.

## 0. 타깃 한눈에

| | 🅰 김총대 | 🅱 이아무 | 🅲 박모임 |
|---|---|---|---|
| 역할 | **방장 / 결정 주도자** | **참여자 / 묻어가는 사람** | **정기 모임 운영자** |
| 우선순위 | **P0 (1순위)** | **P0 (1순위)** | P1 (확장·리텐션) |
| 한 줄 | "내가 또 정해?" | "가입 말고 링크만 줘" | "매주 같은 고민 그만" |
| 핵심 가치 | 후보 찾는 노동 0 | 마찰 없는 의견 표출 | 기록 기반 재추천 |
| MVP가 겨냥 | 방 생성 + 자동 후보 | 게스트 입장 + 투표 | (P1) 개인화·이력 |

> **설계 원칙:** P0 데모/MVP는 🅰가 방을 만들고 🅱들이 우르르 들어와 같이 투표하는 **일회성 빠른 결정**에 100% 집중. 🅲(반복·개인화)는 리텐션 지표의 주인공이지만 구현은 P1로 미룬다. → 이 우선순위가 KPI(④⑤)와 설문(⑥) 표본 설계의 기준.

## 🅰 김총대 (28) — 1순위, "맨날 정하는 사람"

**한 줄 정의:** 팀의 점심·회식을 늘 총대 메고 정하다 지친, 결정 주도자.

| 항목 | 내용 |
|---|---|
| 인구통계 | 28세, 판교 50인 SaaS 스타트업 백엔드 3년차, 분당 자취 |
| 디지털 습관 | 얼리어답터. 카톡 단톡 5개+, 네이버지도 즐겨찾기 30곳, 캐치테이블·망고플레이트 가끔. 새 웹서비스 거부감 없음 |
| 결정 빈도 | 팀 점심 **주 5회** + 부서 회식·팀 디너 월 2~3회 |
| 현재 대안 & 한계 | 12시쯤 단톡에 식당 2~3개 링크 직접 붙임 → 반응 없으면 늘 가던 김치찌개집. **후보 수집이 매번 본인 노동**, 반응 안 모이면 교착 |
| 트리거 (모디를 켜는 순간) | 동료 4명 모였는데 아무도 안 정할 때 — "이걸로 정하자" |
| 좌절 트리거 | "아무거나/다 좋아", 정하다 점심시간 까먹음, 못 먹는 사람 뒤늦게 등장 |

**JTBD (Jobs To Be Done)**
- *기능적:* 모인 사람들이 **3분 안에** 갈 식당 한 곳에 합의하게 한다.
- *감정적:* "또 나만 정한다"는 **부담·피로에서 벗어난다**.
- *사회적:* 매번 떠넘기는 사람이 아니라 **"빠르게 정리해주는 사람"** 으로 보이고 싶다.

**성공의 정의:** 5분 안에 '안 가본 집' 한 곳이 **모두 동의로** 정해진다.
**우려(anti-goal):** 새 도구 익히는 데 시간 쓰기 싫다 / 남들이 안 따라오면 헛수고.
**대표 인용:** *"내가 또 정해? 그냥 다 같이 30초만 눌러줘."*
**모디가 주는 것:** 방 생성 즉시 AI 후보 5개 → 후보 수집 노동 제거. 링크 한 번으로 팀을 투표 화면에 모음.

## 🅱 이아무 (24) — 1순위, "아무거나파 참여자"

**한 줄 정의:** 의견은 있지만(특히 '싫은 것') 마찰 있으면 그냥 묻어가는 참여자.

| 항목 | 내용 |
|---|---|
| 인구통계 | 24세, 서울 4년제 경영 3학년, 신촌 자취, 용돈+알바 |
| 디지털 습관 | 인스타 DM·카톡·배달앱 능숙. **새 앱 설치·회원가입에 강한 거부감** ("그걸 굳이 깔아?") |
| 결정 빈도 | 모임 식사 **주 3~4회** (팀플·동아리·친구) |
| 현재 대안 & 한계 | 남이 정하는 대로 따라감. 매운 거 못 먹는 것만 소심하게 말함. **구글폼 '로그인 필요' 뜨면 그냥 이탈** → 내 의견 반영 0 |
| 트리거 | 누가 카톡에 모디 링크 던졌을 때. 누르기만 하면 됨 |
| 좌절 트리거 | 가입·로그인 벽, 긴 단톡 스크롤, 내 "못 먹는 것"이 무시됨 |

**JTBD**
- *기능적:* **설치·가입 없이** 내 선호(특히 비선호)를 한 번에 표시한다.
- *감정적:* 까다로운 사람처럼 보이지 않으면서 **"이건 빼줘"를 안전하게** 낸다.
- *사회적:* 모임에 **민폐 없이 참여**하고, 결정에 한 표 보탰다는 소속감.

**성공의 정의:** 가입 없이 **10초 만에** 입장해 '이건 빼줘(👎)'를 표시.
**우려:** 또 뭐 깔라고? / 내 번호·계정 정보 안 주고 싶다.
**대표 인용:** *"가입 말고, 링크 주면 바로 누를게."*
**모디가 주는 것:** 링크 클릭 → 닉네임만 → 즉시 투표. 👍/👎로 비선호도 부담 없이 표현.

## 🅲 박모임 (31) — P1 확장, "정기 모임 운영자"

**한 줄 정의:** 매주 같은 결정을 반복하는, 기록과 재추천이 필요한 운영자.

| 항목 | 내용 |
|---|---|
| 인구통계 | 31세 직장인. 주말 등산 동호회(20명) 총무 + 평일 사이드 스터디(6명) 운영 |
| 디지털 습관 | 네이버밴드·카톡투표 능숙. 지난 장소 엑셀 정리하려다 귀찮아 포기 |
| 결정 빈도 | 정기 모임 **주 2회** (반복적·예측 가능) |
| 현재 대안 & 한계 | 매주 본인이 후보 재검색→밴드 투표. **"저번에 간 데 어디였지?" 아무도 기억 못 함** → 또 같은 곳 |
| 트리거 | 다음 모임 공지 올릴 때 — "이번 주 어디서?" |
| 좌절 트리거 | 늘 가던 데로 수렴, 매번 새 후보 발굴 노동, 기록 0에서 시작 |

**JTBD**
- *기능적:* 지난 결정을 기억해 **'새롭지만 검증된' 후보**를 매번 자동으로 받는다.
- *감정적:* 매주 반복되는 **운영 피로를 던다**.
- *사회적:* "이 모임 잘 굴러간다"는 **운영자 효능감**.

**성공의 정의:** 지난 기록 기반 추천을 받아 **바로** 투표.
**대표 인용:** *"매주 똑같은 고민 그만하고 싶어."*
**모디가 주는 것 (P1):** 결정 이력 저장 → 개인화 추천("저번엔 한식, 이번엔 다른 거"). → **재사용률**(가장 강한 후행지표)의 주인공.
