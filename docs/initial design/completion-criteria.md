# 4. 완료 기준 (무엇이 되면 끝인가)

> 4단계 프레임 ④. "어떤 게 완료되어야 마무리되는지."
> **이 데모의 완료 기준은 자동 테스트 통과가 아니라, 실제 핵심 기능을 브라우저로 직접 해보면서 문제가 없음을 확인하는 것이다.**

---

## 4.1 완료의 정의 (Definition of Done)

> 아래가 참이면 이 데모는 완료다.

✅ **핵심 동선을 실제 브라우저에서 직접 끝까지 눌러봤을 때, 문제 없이 동작한다.**
(방 생성 → 후보 노출 → 링크 입장 → 실시간 투표 → 확정 — 막힘·깨짐·이상 동작 없음)

자동 테스트(4.3)는 이 수동 확인을 **받쳐주는 보조 수단**이지, 완료의 기준 자체는 아니다.

---

## 4.2 브라우저 실동작 검증 (★ 주된 완료 기준)

실제 기기/브라우저에서 아래를 **눈으로 보며** 확인한다. 각 항목이 기대대로 보이면 통과.

| # | 직접 해보는 것 | 통과 = 브라우저에서 이게 보임 |
|---|---|---|
| V1 | 방장: 조건(강남역·4명·1만원대) 넣고 방 생성 | **체감 3초 내** 후보 5개 카드가 뜬다 |
| V2 | 발급된 링크를 **다른 브라우저/폰**에서 연다 | 가입창 없이 닉네임 입력 화면 → 입장된다 |
| V3 | 참여자가 후보에 좋아요/싫어요 | 누른 표가 **방장·다른 참여자 화면에 1~2초 내** 차오른다 |
| V4 | 같은 후보를 두 번 누른다 / 좋아요↔싫어요 바꾼다 | 표가 누적 안 되고(1로 유지), 변경은 반영된다 |
| V5 | 방장이 결과 확정 | 전원 화면에 결정 카드가 뜬다. 방장 아닌 사람은 확정 버튼이 없다/막힌다 |
| V6 | 1명만 있는 방 / 아무도 투표 안 한 방 | 화면이 깨지지 않고 안내가 보인다 |
| V7 | 참여자가 후보 직접 추가 | 추가한 후보가 모두에게 보인다 |
| V8 | 폰 브라우저(모바일 폭)에서 전체 동선 | 레이아웃 안 깨지고 다 눌러진다 |

### 검증 방식
- **로컬 멀티 세션:** 일반 창 + 시크릿 창(또는 브라우저 2개)으로 방장/참여자를 동시에 띄워 V1~V7 확인.
- **실배포 멀티폰:** Vercel 링크를 **실제 폰 2대 이상**(셀룰러 포함)에서 열어 V2·V3·V5를 한 번에 시연 → 이게 되면 데모 준비 끝.
- (선택) 이 워크스루를 GIF/영상으로 남겨두면 회의·포폴에 그대로 쓸 수 있다.

> 핵심: **"테스트가 초록불"이 아니라 "내가 브라우저로 해봤더니 된다"** 가 완료의 도장.

## 4.3 자동 테스트 (보조 — 회귀 안전망)

수동 확인을 매번 반복하지 않도록 받쳐주는 안전망. 데모라 과하게 깔지 않는다.

| 계층 | 대상 | 도구 | 핵심 케이스 |
|---|---|---|---|
| 단위 | `core/domain`, `core/application` | Vitest | 집계 점수·랭킹, 중복투표 upsert(V4), 확정 권한(V5) |
| 통합 | API Route Handlers + `InMemoryRoomRepository` | Vitest | create→join→vote→state→decide 플로우, 권한 거부, 빈 방(V6) |
| E2E(선택) | 브라우저 멀티 세션 | Playwright | 4.2의 V1·V3·V5를 자동 재현 — 수동 검증을 코드로 박제 |

> 구조 보상: core가 인프라 비의존이라 `InMemoryRoomRepository`+`MockCandidateProvider`+가짜 `Clock`으로 단위·통합 테스트가 빠르고 결정적이다. E2E는 시간이 남으면 V1·V3·V5만 우선 자동화한다.

## 4.4 기능 수용 기준 (Given–When–Then, 4.2의 근거)

V1~V7이 "무엇을 보면 통과인지"의 정밀 정의.

1. **후보 자동 생성 (V1):** Given 방장이 "강남역·4명·1만원대"로 방을 만들면 / When 생성을 누르면 / Then 체감 3초 내 후보 5개가 카드로 뜬다.
2. **가입 없는 입장 (V2):** Given 참여자가 링크를 열면 / When 닉네임을 입력하면 / Then 로그인 없이 입장해 투표할 수 있다.
3. **실시간 집계 (V3):** Given 한 참여자가 투표하면 / Then 모든 참여자 화면에 1~2초 내 집계가 갱신된다.
4. **중복 방지 (V4):** Given 같은 후보에 좋아요 2번 / Then 표는 1로 유지, 좋아요↔싫어요는 변경된다.
5. **확정 (V5):** Given 방장이 확정하면 / Then 상태가 `decided`로 바뀌고 전원에 결정 카드 표시. 방장 아닌 사람은 확정 불가.
6. **엣지 (V6·V7):** 빈 방에서도 안 깨지고 안내된다. 참여자가 추가한 후보가 모두에게 보인다.
