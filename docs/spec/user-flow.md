# 사용자 흐름 — 모디

## 상태 기계 (방 = `/room/[id]` 한 라우트가 상태로 변신)
```
waiting ──[방장: 투표 시작]──▶ voting ──[타이머 0 | 방장: 지금 마감]──▶ decided
(생성·대기)                   (타이머 가동)                          (1등 자동 공개)
```
- **라우트:** `/`(랜딩) · `/create`(생성 폼) · `/room/[id]`(역할×상태로 화면 전환, 폴링 대상).
- **역할:** 방장 = 투표자 + 운영자(화면 = 참여자 화면 + 운영 버튼). 참여자 = 링크+닉네임.

## 타이머 규칙 (의도: 데모 클라이맥스를 방장이 무대에서 터뜨림)
- **시작:** 방장이 `[투표 시작]`을 누를 때(생성 즉시 아님 — 링크 먼저 뿌리고 사람 모인 뒤). 첫 참여자 대기 동안 시간 안 깎임.
- **길이:** 생성 시 프리셋 **1/3/5분**(기본 3, 데모 1).
- **0초:** 자동 마감 + 그 시점 1등 자동 공개. 방장은 `[지금 마감]`으로 조기 종료 가능.
- **동점:** 점수 같으면 **그 점수에 먼저 도달한 후보** 승(`votes`의 seq로 판정).
- **구현:** `deadlineAt` 저장 → 폴링 시 지났으면 lazy 자동 확정(크론 불필요).

## 해피패스
```
방장:  랜딩→만들기 → 폼(모임명·강남역·인원·예산·타이머) → 생성(1.5s "후보 뽑는 중")
       → 방(대기): 후보5 + [URL 복사] + [투표 시작]
참여자: 링크 → 닉네임(가입X) → 대기 라운지 → (방장 시작) → 투표
투표:  후보 👍/👎. 내 표 즉시(낙관적), 남의 표 1초마다 카운트업. [+후보 추가] 가능
결과:  타이머0/지금마감 → 잠김 → 🎉컨페티 + 1등 카드 전원 표시
```

## 화면별 구성 (그릴 수 있는 수준)

**대기 라운지(waiting)** — 후보 카드 미리보기(👍/👎 **비활성**) + "방장이 곧 시작해요" + `👥 N명 모이는 중`(실시간↑). 방장 화면 하단 `[투표 시작]`. *의도: "사람이 모인다"를 눈으로 — 아하②의 감정.*

**투표(voting)**
```
┌ 점심 모임 🍜      👥 4   ⏱ 00:47 ┐   ← 헤더 고정. 타이머 막판 10초 빨강+깜빡
│ ┌ 🍜 강남 손칼국수  [안가본집] ┐  │
│ │ 칼칼한 국물 · 9,000원~        │  │
│ │   👍 3          👎 1          │  │   ← 숫자만(아바타 X)
│ │ [ 좋아요 ]   [ 별로예요 ]      │  │   ← 3상태: 없음/👍채움/👎채움, 재클릭=취소, 반대=전환
│ └────────────────────────────┘  │   ← 후보마다 독립, 여러 후보 동시 가능
│ [+ 후보 직접 추가]                │
├──────────────────────────────────┤
│ 방장만: [ 지금 마감하기 ]          │   ← 푸터 고정(대기땐 [투표 시작]), 참여자엔 안 보임
└──────────────────────────────────┘
```

**결과(decided)** — `canvas-confetti` + 1등 카드 zoom-in: "결정됐어요! / 식당명 · 설명 · 👍N 최다득표" + `[처음으로]`. 지도 미사용 → **길찾기 버튼 없음**.

## 엣지
빈 방(1명/0표)도 안 깨지고 안내 · 닉네임 중복 허용 · 후보 추가는 대기·투표 중 모두 가능 · 한 폰 돌려쓰면 같은 사람 취급(데모 허용).
