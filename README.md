# 모디 (MODI)

> **"오늘 점심 뭐 먹지?"를 3분 안에 끝내는 AI 실시간 그룹 결정 데모.**
> 방을 만들면 후보가 자동으로 뜨고, 친구들이 링크로 들어와 실시간으로 같이 투표해 즉석에서 정한다.

이 저장소는 **스위프 웹 14기 4조 1주차 회의에 가져갈 "보여주기식" 데모 앱**이다.
실서비스가 아니라 **컨셉을 실제로 눌러보게 만드는 것**이 목적이다.

---

## 한눈에 보는 결정

| 항목 | 결정 | 한 줄 이유 |
|---|---|---|
| 형태 | Next.js 풀스택(App Router), Vercel 배포 | 회의용 데모를 가장 빠르게 배포 |
| 코어 구조 | 헥사고날(포트 & 어댑터) | 나중에 코어를 별도 서버(Spring 등)로 자연스럽게 분리 |
| 저장소 | Upstash Redis (= 관리형 인메모리 DB) | "메모리 DB" 제약 충족 + 서버리스 인스턴스 간 상태 공유 |
| 실시간 | 1초 폴링 | 서버리스에 가장 잘 맞고, 데모에선 "실시간"으로 충분 |
| 후보 생성 | **mock 데이터** (LLM 호출 0) | 비용·키 없이 동일한 화면 연출 |
| 외부 API | 유료·복잡 인증 회피 | Upstash 무료 티어(카드 X) 외 외부 의존 0 |
| UI/디자인 | Tailwind + shadcn/ui, 웜 톤(에스프레소·크림·테라코타) | 카드 중심 모바일 데모를 빠르고 예쁘게 |
| 진행 방식 | 방장이 [투표 시작] → 타이머(1/3/5분) → 0초 자동 우승 공개 | 데모 클라이맥스 연출 |
| 공유 | URL 복사만 (카카오 SDK X) | 복잡 인증 회피 |

자세한 근거는 [`docs/spec/decisions.md`](docs/spec/decisions.md).

## 설계 문서

### ⭐ 정본 — `docs/spec/` (AI 에이전트·개발자 단일 참조)
구현 중 참조하는 확정본. 짧게, 의도/맥락은 남기는 원칙으로 작성.

1. **[`spec/product.md`](docs/spec/product.md)** — 제품 기능 정의서 (성격·문제·아하·페르소나·P0·비목표·성공)
2. **[`spec/user-flow.md`](docs/spec/user-flow.md)** — 사용자 흐름 (상태기계·타이머·화면별 구성·엣지)
3. **[`spec/data-schema.md`](docs/spec/data-schema.md)** — 데이터 스키마 (`room:{id}` JSON·집계규칙·신원)
4. **[`spec/architecture.md`](docs/spec/architecture.md)** — 프로젝트 아키텍처 (스택·헥사고날·포트·API 7개·미래분리)
5. **[`spec/decisions.md`](docs/spec/decisions.md)** — 기술적 결정사항 (결정·왜·트레이드오프·제약·가드레일)

> **AI/개발자는 `docs/spec/`만 읽으면 된다.**

### 작업 기록 (참고용, 정본 아님)
- `docs/initial design/` — 초기 4단계 설계 스냅샷
- `docs/detailed design/` — 9단계 세부 설계 과정(와이어프레임 원본 포함)

> 원본 아이디어 제안서(서병필 팀장): 4조 모디(MODI) PRD. 이 문서들은 그 PRD를 **"회의용 데모"로 좁혀** 재구성한 것이다.
